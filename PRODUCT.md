# Product

## Register

product

## Users

独立开发者，日常需要管理多个 API 密钥（OpenAI、Claude 等）。场景：在本地环境中快速添加、分组、测试和清理 API 密钥，追求效率和信息密度，不想要花哨的东西。

## Product Purpose

一个本地运行的 API 密钥管理工具。核心工作流：批量导入密钥 → 按服务商分组 → 一键测试可用性 → 剔除失效密钥。成功标准是开发者能在几秒内完成密钥管理操作，不需要思考工具本身。

## Brand Personality

专业、高效、可靠。像一个安静的终端工具——存在感低但随时可用。不炫耀，不打扰，把事情做完。

## Anti-references

- Web3 / 加密货币工具风格：霓虹色、暗黑渐变、过度动效
- 企业后台：微软/谷歌式的灰色表格、死板的表单布局
- 典型 SaaS dashboard：花哨的渐变、大圆角卡片、过度留白

以上风格的共同问题是过度设计或过度保守，都不适合一个开发者自用的本地工具。

## Design Principles

1. **效率优先**：信息密度高于呼吸感。密钥管理是高频操作，每一个多余的像素都是摩擦。
2. **安静的存在感**：工具应该像终端一样——用完即走，不试图吸引注意力。
3. **信任但验证**：界面传达可靠感，测试结果一目了然，状态清晰可信。
4. **不做多余的事**：不加没有明确需求的功能、装饰、动效。简洁不是风格，是纪律。

## Accessibility & Inclusion

基础无障碍：颜色对比度足够（WCAG AA 色值对比度），核心操作可键盘完成，色盲友好的状态指示（不依赖单一颜色区分状态）。
